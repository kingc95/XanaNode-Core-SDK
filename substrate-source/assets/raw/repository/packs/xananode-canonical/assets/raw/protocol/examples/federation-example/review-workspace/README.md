# Federation Review Workspace

This example is the temporary review view a XanaNode tool can expose after mounting both federation example substrates together.

- It keeps `example.researcher_a:concept/knowledge-substrate` and `example.researcher_b:concept/memory-substrate` as separate nodes.
- It makes their `possible_match` overlap visible instead of silently collapsing them.
- It adds a short trail and review concept so the mounted result can be opened as a real substrate, not only read as reports.

This is not the only possible review projection. It is one valid example of the non-destructive merge state that sits between "independently authored substrates" and "human-approved permanent merge."
